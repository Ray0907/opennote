## Priority Issues (remaining)
[P1] Table row delete unguarded — DatabaseView.onDeleteRow calls deletePage immediately, no confirm/undo, unlike the sidebar delete it mirrors. Fix: route row deletes through the same undo-Toast path.
[P1] Hover-only actions keyboard-invisible outside the tree — :focus-within only on .tree-actions. .page-header-controls, .page-cover-actions stay opacity:0 hover-only (focus ring on invisible target); table title ↗ open is visibility:hidden (not tabbable). Fix: :focus-within {opacity:1} on those groups; swap ↗ to opacity/focus-within reveal.
[P2] Online sync dot pulses Working Blue every ~3s forever while typing — tick() emits syncing unconditionally; .sync-dot.syncing runs sync-pulse in --accent. Collides with anti-reference (animate while typing) + Working Blue Rule. Fix: only enter syncing when outbox non-empty.
[P2] SearchDialog asserts aria-modal without a focus trap or focus restore — Tab escapes to background, unlike TrashDialog's native trap. Fix: trap Tab + restore focus, or use native <dialog>.
[P2] <select multiple> cells break table density (40-64px in a 2px/6px grid). Fix: chip-summary cell that opens on click/focus.
[P3] .sidebar scrolls whole aside (header/footer scroll away); overflow should be on .tree with min-height:0.
[P3] No narrow-window (~800px) sidebar-collapse affordance.
[P3] No "reveal vault in Finder" affordance for a data-sovereignty product.
