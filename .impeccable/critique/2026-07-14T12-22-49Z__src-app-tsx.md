## Anti-Patterns Verdict: NOT AI slop
LLM: passes all DON'Ts; empty light screen renders zero blue pixels (Working Blue Rule holds). Failures are unfinished-engineering tells (window.prompt), not AI-generation tells. Faithful-Notion familiarity is intended identity. Only scent: COVERS `plum` gradient, but it's opt-in content.
Detector: 12 findings, exit 2, all advisory. Real drift = 0 — 10 colors are COVERS page-cover art (domain content), 2 font-sizes are emoji glyph sizes (56px page-icon, 22px picker). All intentional.
Browser overlay: not available this session; verified via out-of-band light/dark screenshots. Dark mode is a real port, not a filter.

## What's Working
1. Dark theme is a real port — semantic tokens, warm hairline #33322f, dark-deepened shadows, muted ink raised to >=4.9:1, prefers-reduced-motion fully suppressed.
2. Accent discipline survives implementation — zero blue on empty screen; selection is tone shift not blue fill (Two-Layer / Working Blue rules).
3. Search palette is the right pattern done right — 120ms debounce, arrow/Enter/Escape, active hit takes Desk tone, shadow only on floating layer, focus into input on open.

## Priority Issues
[P0] window.prompt() property wizard (DatabaseView.tsx:67-119) — up to 4 OS prompts for a rollup, typed type selection, silent abort on typo. Fix: anchored popover with name field + native select + conditional rows + inline validation.
[P0] Unguarded delete with invisible trash (Sidebar.tsx:70-78, App.tsx:127-139) — immediate destruction of page + vault .md from a hover target; trash exists in data not UI. Fix: undo toast + Trash entry in sidebar footer, non-modal.
[P1] No sync/offline/save indicator — violates Principle 4; data-sovereignty audience needs proof-of-write. Fix: sidebar footer micro line with synced/syncing/offline states.
[P1] Editor typography breaks One Voice + title/body gutter misaligns — BlockNote imports Inter + own padding; 34px title ~52px left of prose column. Fix: override .bn-editor font-family, align BlockNote inline padding with .page-title.
[P2] color-scheme not tied to manual theme override (styles.css:56-58) — html{color-scheme:light dark} follows OS; forcing light on dark OS gives dark native controls in a light app. Fix: :root{color-scheme:light} + :root[data-theme='dark']{color-scheme:dark}.

## Persona Red Flags
Alex (power user): no New Page/DB shortcut; no context menu; Cmd+K finds but never creates; 4-prompt wizard.
Jordan (first-timer): +DB/◐ opaque; hover-hidden actions undiscoverable; first delete is a betrayal; Templates menu opens above footer like a glitch.
Sam (keyboard/a11y): tree-item is div onClick — no tabIndex/role, no keyboard page selection except Cmd+K; hover actions have no :focus-within reveal — unreachable by keyboard; icon buttons title-only no aria-label; SearchDialog no role=dialog/focus trap; light --muted-ink #9b9a97 on white ~2.9:1 fails WCAG AA at 13px (dark's 4.9:1 passes).

## Minor Observations
- .sidebar overflow-y scrolls whole aside; scroll should live on .tree.
- <select multiple> (40-64px) destroys table 2px/6px density.
- icon/cover pickers have no outside-click/Escape dismiss.
- disabled Export relies on title tooltip (unreachable — disabled buttons not focusable).
- database page reuses 820px cap; multi-property tables feel cramped.

## Questions to Consider
1. The vault is the soul of the product — why can't I see it? No path, no Reveal in Finder, no proof the .md exists.
2. Is sync state global or per-page? Footer dot vs per-page glyph — which does a 2-10 person team need during conflict?
3. At ~800px, whose column shrinks first? No collapse affordance despite PRODUCT.md's narrow-window requirement.
4. If earned familiarity is the brand, is BlockNote's default chrome (Inter, gutters, slash menu) inside or outside the brand?
